Javascript: Eventi e metodi

  • SirioLib.getComponentById(id: string, type: string): restituisce l'oggetto Sirio Component del tipo richiesto tramite ID se esiste. Altrimenti genera un'eccezione. Parametri: id: stringa dell'identificativo del componente; type: stringa del tipo di componente, in questo caso 'dropdown-menu'.
  • getMenuElement(): restituisce l'oggetto menu HTMLElement collegato all'oggetto SirioDropdownMenu chiamante.
  • getTriggerElement(): restituisce l'oggetto trigger HTMLElement collegato all'oggetto SirioDropdownMenu chiamante.
  • getItemElements(): restituisce gli oggetti item HTMLElement collegati all'oggetto SirioDropdownMenu chiamante.
  • isHidden(): restituisce true se il menu a discesa è nascosto. In caso contrario, restituisce false.
  • show(): mostra il Dropdown Menu.
  • hide(): nasconde il Dropdown Menu.
  • toggle(): mostra o nasconde il Dropdown Menu, a seconda del suo stato attuale (mostrato o nascosto).

Card

Il titolo di ogni card va conformato con il livello di HEADING in corso (h2, h3, h4,...)

Con l'aggiunta della classe sirio-card-dark alla card si ottiene il colore scuro. In questo caso cambiare la classe dei bottoni usati con sirio-btn-tertiary in sirio-btn-tertiary-light e aggiungerla ai bottoni con classe sirio-btn-icon.

Se usati all'interno del System Grid di Bootstrap, per ottenere che le card siano della stessa altezza aggiungere la class h-100 alla card.

Javascript: Eventi e metodi

  • SirioLib.getComponentById(id: string, type: string): restituisce l'oggetto Sirio Component del tipo richiesto tramite ID se esiste. Altrimenti genera un'eccezione. Parametri: id: stringa dell'identificativo del componente; type: stringa del tipo di componente, in questo caso 'dialog'.
  • SirioLib.SirioDialog(id: string): crea il componente Dialog con id dell'elemento HTML avente attributo data-sirio-component="dialog". Utile nel caso si voglia creare una finestra di dialogo lato javascript senza avere a disposizione un bottone di attivazione. (Si rimanda all'esempio di utilizzo).
  • getDialogElement: restituisce l'oggetto dialog HTMLElement collegato all'oggetto SirioDialog chiamante.
  • getTriggerElements: restituisce gli oggetti trigger HTMLElement collegati all'oggetto SirioDialog chiamante.
  • getDismissElements: restituisce gli oggetti HTMLElement scartati collegati all'oggetto SirioDialog chiamante.
  • setOnShow(callback): imposta la funzione di callback da eseguire quando la finestra di dialogo sta per essere visualizzata.
  • setOnShown(callback): imposta la funzione di callback da eseguire quando viene visualizzata la finestra di dialogo.
  • setOnHide(callback): imposta la funzione di callback da eseguire quando la finestra di dialogo sta per essere nascosta.
  • setOnHidden(callback): imposta la funzione di callback da eseguire quando la finestra di dialogo è stata nascosta.
  • show(): mostra la finestra di dialogo.
  • hide(): nasconde la finestra di dialogo.
  • toggle(): mostra o nasconde la finestra di dialogo, a seconda del suo stato attuale (mostrato o nascosto).

Constructor SirioToast(toastEl: HTMLElement, onShow?: VoidFunction, onHide?: VoidFunction, onShown?: VoidFunction, onHidden?: VoidFunction).
Parameters:

Eventi e metodi:

  • getToastEl(): restituisce l'oggetto HTMLElement del componente toast collegato all'oggetto SirioToast chiamante.

  • getToastType(): restituisce la stringa di tipo toast collegata all'oggetto SirioToast chiamante.
  • getDismissDelay(): restituisce il ritardo(ms) collegato al timeout dell'oggetto SirioToast chiamante prima dell'auto-hiding.
  • isHidden(): restituisce true se l'oggetto HTMLElement del componente toast collegato all'oggetto SirioToast chiamante è chiuso, altrimenti restituisce false.
  • shown(): mostra l'oggetto HTMLElement del componente toast collegato all'oggetto SirioToast chiamante.
  • hide(): nasconde l'oggetto HTMLElement del componente toast collegato all'oggetto SirioToast chiamante.
  • dispose(): distrugge l'oggetto HTMLElement del componente toast collegato all'oggetto SirioToast chiamante.
  • setDismissDelay(delay: ritardo da impostare): il ritardo (ms) collegato al timeout dell'oggetto SirioToast chiamante prima dell'auto-hiding.
  • setOnShow(callback): imposta la funzione di callback da eseguire quando Toast sta per essere mostrata.
  • setOnShown(callback): imposta la funzione di callback da eseguire quando Toast è stata mostrata.
  • setOnHide(callback): imposta la funzione di callback da eseguire quando Toast sta per essere nascosta.
  • setOnHidden(callback): imposta la funzione di callback da eseguire quando Toast è stata nascosta.

Avvisi

L'Avviso è un elemento che permette di mostrare una comunicazione. Comunica all’utente un cambiamento, una novità su una sezione, una campagna informativa, un nuovo contenuto o aggiornato che abbia una natura di interesse primario e di rilevanza importante. Un Avviso dovrebbe essere posizionato immediatamente dopo la sezione hero.

Nel paragrafo descrittivo è possibile inserire un hyperlink che permette di accedere ad un eventuale sezione di approfondimento.

Il bottone di azione è opzionale, deve essere utilizzato quando è necessario veicolare l’utente su una sezione specifica riferita al titolo.

Si sconsiglia l’inserimento di più Avvisi nella stessa pagina, qualora fosse necessario si suggerisce di inserirne un massimo di due in righe successive

ATTENZIONE: il componente nasce per essere inserito all'interno di una struttura ben precisa, che potete copiare direttamente dal codice sottostante gli esempi, contenente colore di sfondo, bordo inferiore e dimensione massima orizzontale.
